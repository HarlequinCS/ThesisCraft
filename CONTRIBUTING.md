# Contributing

Contributions are welcome.

## Before you change anything

1. Read `README.md`.
2. Read `skills/humanizer/SKILL.md`.
3. Check whether the change preserves meaning, citations, and technical terms.
4. Make sure the output still sounds natural and does not add unsupported claims.

## What to contribute

- Better AI pattern rules
- Better academic safeguards
- Better examples
- Better installation instructions
- Bug fixes in the skill wording

## What to avoid

- Adding fabricated research claims
- Removing citations that belong to the source text
- Making the skill more verbose without reason
- Introducing em dashes into the skill or examples

## Pull request notes

Please describe:

- what changed
- why it changed
- whether the change affects citation handling
- whether the change affects academic writing support

## Style

- Use plain, direct English.
- Keep the skill files readable.
- Prefer small edits over large rewrites unless the change is clearly better.
